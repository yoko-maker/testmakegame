# 🔒 PAIR LOCK — 二人協力・非対称情報の暗号脱出

「二人とも主人公」ではなく「**二人とも不完全**」。
各プレイヤーは半分の情報しか持たず、声・チャットで補い合わないと絶対に解けない協力謎解き。

## 遊び方（2人 / 別端末）

1. 1人が **ルームを作成** → 表示された4文字のコードを相手に伝える（= P1）
2. もう1人が別の端末で **ルームに参加** にコードを入力（= P2）
3. 2人揃うと Stage1 開始。各端末には情報の半分しか出ないので、**見たものを言葉で描写**し合って解く
4. Stage1〜4 → Final（中央制御室）を突破で脱出

- **施設安定度**: 不正解ごとに −5%。0% で封鎖（ゲームオーバー → 再起動可）
- **スタンプ**: 通話できない時の合図（👍OK / 🔁もう一回 / ❓わからない / ✋待って）
- **ランダム生成**: ルームごとに seed が変わり、毎回違う問題になる

## エンディング

| エンド | 条件 |
|---|---|
| ✅ Normal | Final 突破で脱出 |
| 🩸 Secret | Stage4 の「裏ログ」断片を両者で繋いで照合（真相判明） |
| 🌟 True | ノーミス（安定度100%維持）＋ 裏ログ回収 |

## 動作要件・同期の仕組み

- `@st.cache_resource` による**サーバ内共有ストア**でルーム状態を同期し、
  `st.fragment(run_every=...)` のポーリングでリアルタイム反映する。
- **前提: 2人が同一の Streamlit サーバプロセスに接続していること。**
  ローカル起動や単一サーバ運用ならOK。複数レプリカ構成（一部のクラウド）では
  ルームが共有されない場合がある。
- `streamlit >= 1.37`

## 実行

```bash
# 単体
streamlit run app.py

# ポータル経由（リポジトリのルート）
streamlit run app.py   # → ホームから「PAIR LOCK」を選択
```
